1Overview and purpose

Automower Trade-in Form is a web application for the Husqvarna dealer network. It allows authorised dealers to submit trade-in applications for Automower robotic lawnmowers – dealers register a customer's trade-in unit, upload photos and invoices, and receive a confirmation ID for case tracking.

LanguagesEnglish (/en) and Austrian German (/de-at)
Admin accessSSO via Microsoft Entra (Husqvarna internal accounts only)
DatabaseSQLite (primary) + Databricks Delta (sync / analytics)

3Functionality

Dealer form (public)

Dealers fill in a form with:

  • Dealer information: dealer number, company name, location, email
  • Product information: sold model, new serial number, trade-in type, trade-in serial number
  • File uploads: nameplate photo, product image, invoice – max 25 MB per file, formats: .png .jpg .jpeg .pdf
  • Multiple products: support for adding additional products to the same submission

Admin panel (SSO-protected)

  • List and search submissions
  • View images and PDFs (proxied via /api/files/)
  • Export all submissions to Excel (.xlsx) with clickable file links
  • Filter by date range

Databricks integration

Each submission triggers a background task that uploads files to the Databricks Unity Catalog Volume and inserts rows into the Delta Lake tables. If Databricks is not configured or unavailable, SQLite acts as the fallback data store.

n8n webhook (optional)

If the environment variable N8N_WEBHOOK_URL is set, a notification is posted to that URL on each new submission. Errors are logged but do not block the form response.

4API endpoints

Public endpoints

GET /healthz Health check v2 (JSON with buildInfo) – required by DevPlatform
GET /healthz/liveness Liveness probe – returns plain text UP
GET /healthz/readiness Readiness probe – same payload as /healthz
POST /api/submissions Create a new submission (multipart/form-data with JSON payload + files)
GET /en, /de-at Form HTML – directly linkable per language

Admin endpoints (require Entra SSO session)

GET /api/submissions Fetch submissions (from Databricks, fallback SQLite). Params: limit, offset
GET /api/submissions/flat Flat rows per product – for analytics. Params: from_date, to_date
GET /api/submissions/export Download all submissions as Excel (.xlsx) with file hyperlinks
GET /api/files/{path} Proxy a file from the Databricks Unity Catalog Volume to the browser
GET /api/admin/databricks-status Diagnostics – tests Databricks connectivity and returns table names

6Deployment – Husqvarna DevPlatform

Platform deviation This app uses Python / FastAPI / REST – not the platform's standard stack (Node / Express / GraphQL). It runs as a standalone container on OpenShift with no GraphQL subgraph or Hive registration.

8Local development

Option 1 – Docker Compose

docker compose up --build

The app starts at http://localhost:4001.

Option 2 – uvicorn directly

# Create .env in the project root (gitignored) with real values
cp .env.example .env   # edit as needed

# Install dependencies
pip install -r requirements.txt

# Start
uvicorn backend.app:app --reload --port 4001

Local .env (root)

The .env file in the project root is gitignored and used only locally. load_dotenv() in app.py reads it as a fallback if environment variables are not already set by the platform.
